# 3/22/2026 3 AM - Initial Design and Component Selection

_Time spent: 3.5h_

One day, I found myself very intrigued by this video from YouTube creator Mitxela (https://www.youtube.com/watch?v=CHoGIvOi-jw) and decided that making some cool matrix earrings would be a fun way to master my skills in the miniaturisation of electronics.

![image](https://stasis.hackclub-assets.com/images/1774149397835-czvpk0.png)
*image credit: mitxela*

However, I wanted my design to be completely custom, and that includes the enclosure. More importantly, I also wanted my design to be as small as possible (less than 10mmx10mm) while still integrating a microcontroller and *some* sensor so that the studs would react to changes in their environment. 

Since these earrings would be so small, I was also prepared to make sacrifices on the actual LED matrix, as having an especially high-resolution matrix in an area less than 1cm^2 would be a waste of power and introduce significant complexity (Also, having an especially dense board would require a lot of vias, which take up additional space as JLCPCB can't manufacture boards with blind or buried vias.

The third unique thing I wanted to have in my design was a rechargeable Li-ion battery, the same as the ones used in wireless earbuds. This also means that I'll have to create a battery charging circuit for the MatrixStuds, probably in a sort of case.

So, with my ideation complete, I set out on a hunt for components that would fit my extremely constrained dimensional requirements. Researching on various forums and articles, I found some strong contenders for what microcontroller I would include. Since this microcontroller would have to have enough memory to load in various animations, and enough computing power to run some sort of algorithm to control the matrix using sensor input, have enough GPIO pins to drive a reasonabler matrix, and consume a small enough emount of power to last at least a whole day from a tiny 25mAh battery, this narrowed the search down significantly. At the end, I settled on the MAX32660 microcontroller, which is quite computationally beefy for its tiny 3x3mm size desppite being one of the most efficient MCUs I could find.

![image](https://stasis.hackclub-assets.com/images/1774149292266-i2mojm.png)

After that, I started looking for some sensor to put in the earrings so that it would actually feel "alive". Initially, I thought about using a light sensor, or a very small microphone to respond to voice, but in the end I chose to go with a ridculously tiny 3-axis accelerometer by Bosch: the BMA530. This sensor is not far from the size of an 0201 resistor, measuring only 1.2mm on its longest axis. This would free up a ton of space on the board, and since it uses Bosch's very easy-to-use Sensor API, it can interface with the microcontroller effortlesly.

![image](https://stasis.hackclub-assets.com/images/1774149333419-e5vnmi.png)

As the MAX32660 cannot handle any supply voltages above 3.6V (As it is probably designed to be powered by 3V coin cells), I would have to take the 3.7V nominal voltage from a Li-Po battery and step it down to 3.3V for the components. For this, I chose the XCL202B331ER-G by TOREX for its 2.5x2mm footprint. As for the rest of the components, I will be using 0201 passives where possible, and the LEDs will be 0201-sized in a cool white colour. 

As the MAX32660 package I selected has around 9 usable GPIOs, I will connect the chip to a 4x4 matrix. Yes, I did consider Charlieplexing for way more LEDs, but that would've been a pain to arrange and code for, and would've also increased ther size of the studs.

Since the component list for this thing is so tiny, I can jump straight into modeling the PCB!
